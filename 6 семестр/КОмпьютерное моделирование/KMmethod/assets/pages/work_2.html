Цель работы

Создание базы данных переменных, участвующих в АСУ ТП.

Задание

Составить список переменных и описать в генераторе базы данных SCADA–системы «КРУГ-2000».

Настроить систему

При создании БД в КРУГ-2000 генерируются несколько таблиц, характеризующих нашу модель: переменные, каналы, абоненты, самописцы и другие. В процессе данной работы необходимо заполнить эти таблицы согласно проектируемому техническому процессу.

Описание базы данных стоит начать с основных настроек системы. Для этого необходимо задать её название и количество переменных каждого типа.

Добавить и описать переменные

Далее необходимо добавить переменные, спроектированные в прошлой работе и описать их. На данный момент настраивать условия и правила взаимодействия между переменными не нужно. Однако предупредительные и предаварийные границы при необходимости всё же указываются. Остальные условия и границы будут задаваться в следующей работе.

Здесь описаны параметры для каждого типа переменных, которые понадобятся для реализации заданной системы. Остальные параметры с высокой вероятностью будут избыточны. Подробнее о них можно прочитать в разделе «Помощь» программы Круг-2000.

Кроме основных типов переменных в системе Круг-2000 могут задаваться и переменные ручного ввода. Это вещественные, логические или строковые переменные, значения которых вводятся и изменяются пользователем вручную в процессе моделирования.

Добавить и описать абонентов

Абоненты — элементы системы, которые собирают и обрабатывают информацию, осуществляют управление системой.

Для данной учебной задачи нам будет достаточно двух типов абонентов: контроллер и станция оператора. Наша простейшая задача не требует дополнительного описания абонентов.

Обмен данными между абонентами осуществляется по каналам.

После добавления абонентов необходимо описать адаптер (устройство сопряжения абонентов системы). Укажите имя адаптера и IP адрес. Для нашего проекта нужен адрес 127.0.0.1 (стоит по умолчанию).

Добавить и описать каналы

Каналы обеспечивают связь между элементами системы: абонентами, датчиками. Нам необходимо добавить канал для каждой переменной, чтобы передавать значения в контроллер. Эти каналы будут относиться к абоненту «контроллер».

Добавить и описать самописцы и перья

Самописцы и перья необходимы для сбора данных, визуализации и дальнейшего мониторинга технологического процесса.

Самописцы фиксируют и хранят параметры технологического процесса в течение заданного интервала времени с заданной частотой.

Каждый самописец характеризуется пером, каждое из которых собирает информацию о конкретной переменной. Также перья определяют как будут визуализироваться значения переменной на трендах.

Количество самописцев может быть произвольным, однако количество перьев рекомендуется выбирать равным количеству переменных (входных и выходных). в системе КРУГ выделяют базовые и производные самописцы.

Производные самописцы отличаются от базовых тем, что фиксируют значения не через стандартные интервалы, а через пользовательские, которые настраиваются отдельно.

Здесь описаны параметры для самописцев и перьев, которые понадобятся для реализации заданной модели управления. Остальные параметры с высокой вероятностью будут избыточны. Подробнее о них можно прочитать в разделе «Помощь» программы Круг-2000.
