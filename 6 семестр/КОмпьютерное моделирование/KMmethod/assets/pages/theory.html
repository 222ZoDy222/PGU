Генератор БД

Система

Для настройки системы необходимо выбрать соответствующий пункт на панели «Объекты». По стандарту она находится в левой части экрана.

При клике на пункт «Система» откроется панель, в которой необходимо ввести название системы и количество переменных каждого типа (Рис. 1).

Рис. 1.Настройка системы

Переменные

Добавление и описание переменных производится в разделе «Переменные». Для каждого типа выделен свой подраздел (Рис. 2).

Чтобы добавить переменную нужного типа, необходимо раскрыть раздел «Переменные» на панели «Объекты» и выбрать нужный тип. На панели инструментов сверху выберете «Добавить переменную» или «Добавить переменные», если хотите создать сразу несколько.

Описание переменных для всех типов просиходит по одному алгоритму: Выберите настраиваемую переменную в списке по центру экрана, в правой части экрана появится панель-таблица с параметрами переменной.

Рис. 2. Настройка ВД переменных

Абоненты

Добавление и описание абонентов производится в разделе «Абоненты», который находится в разделе «Система» (Рис. 3).

Чтобы добавить абонента, необходимо выбрать раздел «Абоненты» и на панели инструментов сверху выберете выпадающий список «Добавить абонента» и выбрать необходимый тип абонента.

Рис. 3. Настройка абонентов

Каналы

Для добавления каналов необходимо раскрыть список абонентов на панели «Объекты» и выбрать нужного абонента.

В центральной части экрана появится панель, вертикально разделённая на две части(Рис. 4). В нижней части панели нажмите «Добавить канал».

Связь канала с переменной задаётся автоматически на основе параметра «№ канала» переменной. Чтобы убедиться в этом, выберите вкладку с настраиваемым каналом и вкладку «Канальные переменные». В открывшемся разделе выберите вкладку с нужным типом переменной. Если переменная добавилась успешно, вы увидите соответствующую запись.

Рис. 4. Настройка абонентов

Также связь канала с переменными можно увидеть в разделе «Каналы» (Рис. 5).

Рис. 5. Раздел «Каналы»

Чтобы создать произвольный самописец, нужно созадать пользовательский интервал и базовый самописец, а потом связзать их.

Самописцы

Для добавления самописцев необходимо выбрать пункт «Самописцы» на панели «Объекты» или сразу выбрать конкретный тип. Добавить самописец можно, нажав на соответствующую иконку на панели инструментов (Рис. 6).

Рис. 6. Раздел «Самописцы»

Чтобы добавить произвольные самописцы, нужно сначала добавить пользовательские интервалы. Для этого зайдите в соответсвующий раздел внутри пункта «Самописцы» на панели «Объекты»

Далее необходимо добавить перья. Для этого нужно открыть самописец и нажать на соответсвующую иконку на панели инструментов (Рис. 7).

Рис. 7. Раздел «Перья»

Корректировка времени

Для настройки свойств коррекции времени на панели «Объекты» раскрыть пункт «Система» и выбрать «Коррекция времени». При выборе данного пункта на экран вызывается форма с описания параметров коррекции времени.

Для настройки коррекции времени : определите группу корректоров времени, определите свойства корректоров времени, задайте абонентов коррекции времени, задайте параметры переходов Зима/Лето.

Рис. 8.Настройка коррекции времени

КРУГОЛ

Создание проекта

Для создания проекта запустите ИСР КРУГОЛ и в меню на верхней панели управления выберите Файл —> Новый проект

Далее укажите имя проекта и укажите папку, в которой находится файл с БД (.db), выберите платформу «Среда исполнения КРУГОЛ». В учебном проекте остальные поля можно не настраивать.

Создание файла

Для создания файла в меню на верхней панели управления выберите Файл —> Новый —> Программа ФБД или Файл СТ

Чтобы связать код с БД необходимо открыть вашу БД .db через Генератор базы данных, на панели объекты выбрать пункт «Cистема», вкладка КРУГОЛ, добавить программу нажатием на плюс. Далее для добавленной программы указываем имя: название скомпилированного файла .out без расширения.

Синтаксис СТ

В данном пункте рассмотрены только простейшие конструкции, необходимые для решения нашей задачи. Более подробное описание всего языка можно найти в разделе «Помощь».

Язык СТ имеет достаточно стандартный синтаксис. Основной алгоритм (аналог функции main) должен начинаться со строки «НАЧАЛО» и заканчиваться строкой «КОНЕЦ»

Переменные подгружаются из БД и обозначаются, как в Генераторе базы данных: ВАn, ВДn, ДВn, АВn, где n — номер переменной. Также предусмотрено использование констант.

Переменные подгружаются из БД и обозначаются, как в Генераторе базы данных: ВАn, ВДn, ДВn, АВn, где n — номер переменной. Также предусмотрено использование констант.

Арифметические, логические операции и операции отношений обозначаются так же, как и во всех популярных языках программирования. Стоит отметить лишь знаки равенства = и неравенства #. Логические операции обозначаются единичными символами &, |, комментарии двоеточием :Текст комментария.

Основным оператором в нашем коде будет оператор условия. Его синтаксис тоже достаточно стандартен:

ЕСЛИ <условие>
{инструкции и операторы}
ИНАЧЕ
{инструкции и операторы}

Вложенность поддерживается.

Принципы работы ФБД

Программа на языке ФБД представляет собой графическую схему, которая состоит из элементов и связей между ними. Для решения нашей задачи нам понадобится всего 4 типа элементов: переменные, константы, функции, элемент «Если».

Переменные подгружаются из БД и представляют собой блок со входом и выходом. Вход (слева) позволяет записывать значение, а выход (справа) — считывать (Рис. 9). Для добавления переменной на панели инструментов сверху выберите пункт «Список переменных» и в открывшемся окне выберите нужную переменную.

Рис. 9. Схемы элементов «Переменная» и «Константа»

Константы создаются в отдельном окне: на панели со структурой проекта слева выбрать вкладку «Словари» (снизу), в открывшимся разделе выбрать пункт «Глобальные константы». Константы добавляются нажатием знака плюс на верхней панели. Для добавления элемента «Константа» нужно выбрать пункт «Список констант» на панели инструментов сверху и выбрать нужную константу в открывшемся окне. Для констант доступно только чтение (Рис. 9)

В стандартной библиотеке КРУГОЛ содержится множество функций. Нам будут нужны лишь простейшие: арифметические и логические операции, операции сравнения. Элемент функции представляет собой блок с входами (обычно несколькими) и выходами(Рис. 10). Добавить функцию можно, выбрав соответствующий пункт на панели инструментов сверху.

Рис. 10. Схемы элементов «Функции»

Элемент «Если» представляет собой контейнер со входом, разделённый на 2 зоны: если, иначе. В них помещаются другие элементы и цепи. На вход подаётся логическое значение, которое определяет какую часть выполнить(Рис 11.). Добавить элемент если можно, выбрав соответствующий пункт на панели инструментов сверху.

Рис. 11. Схемы элемента «Если»

Чтобы объединить элементы в общий алгоритм, нужно связать их линиями связи. Добавить их можно, выбрав соответствующий пункт на панели инструментов сверху.

Трансляция и отладка

После разработки программы её нужно транслировать. Для этого в меню сверху нужно выбрать пункт «Трансляция». Если программа составлена корректно, в консоль выведется сообщение «Ок. Трансляция прошла успешно», иначе будет указано на ошибки.

Чтобы проводить отладку необходимо запустить Генератор динамики через менеджер задач. После запуска появится возможность построчно проверять правильность работы программы. Настроить и запустить отладку можно, выбрав пункт «Отладка» в меню сверху.

После каждого изменения необходимо перезапускать Генератор динамики и транслировать проект.

Генератор динамики

Менеджер задач

Создать проект можно, запустив менеджер задач и нажав на соответствующую иконку над списком проектов.

Настройка проекта производится в окне «Настройка проекта». Чтобы открыть его, нажмите на кнопку свойства над списком проектов (Рис. 12). Для настройки рабочей директории и параметров запуска нажмите кнопку «Дополнительно».

Рис. 12. Настройка проекта

Настройка конфигурации также производится из окна Настройки проекта. Выберите нужную конфигурацию и нажмите кнопку свойства над списком конфигураций(Рис. 13).

Рис. 13. Настройка конфигурации

Создание мнемосхемы

Мнемосхема создаётся через панель «Окно проекта»(открывается через панель инструментов сверху). Раскройте выпадающий список, откройте контекстное меню ПКМ и в контекстном меню выберите «Новое».

Геометрические примитивы, связи и другие базовые элементы расположены на панели инструментов сверху.

Чтобы тренды корректно отображали данные, необходимо кликнуть по нему ПКМ, в контекстном меню выбрать пункт «Свойства». Во вкладке главная установить тип тренда на «Исторический», создайте и настройте перья в соответствующей вкладке(Рис. 14).

Рис. 14. Настройка тренда

Для добавления элемента из библиотеки откройте вкладку библиотеки на панели Окно проекта, выберите нужную категорию и из открывшегося окна перетащите нужный элемент на мнемосхему.

Изображение настраивается также в окне свойств.

Подключение динамики к элементам

Для добавления динамики к элементам нажмите на нужный элемент ПКМ и выберите «Свойства». Во вкладке «Переменные» добавьте необходимые переменные(Рис. 15).

Рис. 15. Связь между элементом и переменной

Чтобы статические элементы сделать интерактивными, нажмите ПКМ и выберите «Добавить динамику». Теперь в окне свойств появилась вкладка «Реакции», в которой можно настроить реакцию элемента на определённые события(Рис. 16).

Рис. 16. Настройка реакций

Рабочий стол

Рабочий стол создаётся через панель Окно проекта (открывается через панель инструментов сверху). Раскройте выпадающий список, откройте контекстное меню ПКМ и в контекстном меню выберите «Новое».

Настройка рабочего стола происходит в окне «Изменение рабочего стола»: нажмите ПКМ по созданному столу и выберите «Изменить стол». В открывшемся окне выбираем из списка переход при открытии стола (обычно основная мнемосхема), добавьте в содержание стола мнемосхему управления из левого списка, поставьте галочку в пункте «Основной стол»(Рис. 17).

Рис. 17. Настройка рабочего стола

Переходы

Для реализации переходов, нажмите ПКМ на заранее подготовленные элемнты и выберите «Добавить динамику». Теперь в окне свойств появилась вкладка «Реакции». Там нажмите на иконку «Добавить реакцию» и в список назначенных событий перенесите пункт «Нажатие левой клавиши мыши», нажмите кнопку «Готово», из выпадающего списка выберите пункт «Локальный переход» и в пункте «Имя перехода» выберите нужную мнемосхему(Рис. 16).

Отладка системы

Проверка работоспособности построенной модели проводится в режиме имитации. Включить его можно, нажав на соответствующую иконку на панели инструментов сверху.

Чтобы проверить рабочие столы (систему от лица пользователя) нужно запустить приложение «Станция оператора» в менеджере задач.
